# 科研 Skill 实验台设计

## 目标

在现有 FastAPI 网页中验证两项可迁移到团队 Agent Kernel 的能力：多轮科研需求确认，以及仅从指定学术源检索论文。

## 范围

保留现有 Echo、计算器和摘要演示 Skill；新增两个科研 Skill：

1. `research_clarification_skill`：以 `session_id` 保存领域、核心问题、数据/方法、约束和交付物五项状态。每轮只询问一个最高优先级缺失字段，并返回三到四个可选项；字段齐全后输出结构化研究简报和可直接用于检索的查询词。
2. `academic_search_skill`：调用 GitHub 项目 openags/paper-search-mcp 提供的 `paper-search` CLI。固定允许 arXiv、Semantic Scholar、OpenAlex 和 Crossref 四个来源，返回结构化论文列表、各来源状态与失败原因；单个来源失败不阻断其余来源。

不实现登录、数据库、全文下载、综述生成、GitHub 仓库检索或线上代码执行。

## 架构

前端由 `/api/skills` 动态生成可选 Skill；每个调用传入 `session_id`。后端新增会话状态存储和科研 Skill 适配层。路由器在没有模型配置时，根据“研究/课题/论文/文献/检索”等关键词分发到科研 Skill，避免退化成 Echo；有模型配置时仍使用现有模型路由。

学术检索适配层不复刻上游项目的检索逻辑，只负责安全调用其 CLI、限制来源与结果数量、解析 JSON，并把外部命令失败转换为可展示的状态。若 CLI 未安装，接口明确返回安装提示，不伪造结果。

## 交互与错误处理

科研需求确认的首次调用会生成 `session_id`；后续调用必须携带该 ID。未知会话 ID 从空状态开始。用户完成五项信息后，页面展示研究简报与“检索论文”建议。

学术检索最多请求每个来源 5 条结果。429、超时、CLI 缺失和单源错误均作为结构化警告返回，页面仍显示其他成功来源。模型 API 配置继续用于本地实验，但不回显掩码 Key，也不允许任意远程模型地址。

## 验收

1. 新科研 Skill 自动出现在网页 Skill 下拉框中，并可手动调用。
2. 给定同一 `session_id` 的连续五轮输入能填满状态并生成研究简报。
3. 无模型 Key 时，研究和文献检索关键词仍能路由到正确 Skill。
4. 学术检索命令仅收到四个允许来源和受限的结果数；单源失败可返回部分结果。
5. 原有 14 项测试保持通过，新增会话、路由、动态列表和检索适配测试。
